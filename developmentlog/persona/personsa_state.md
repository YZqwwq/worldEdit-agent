# 人格系统参数说明（规则重置与调节依据）

## 规则管理原则

### 1) 可重置

我们支持“人格状态重置”，即恢复默认激素（指标）状态：

- `autonomy_level = 0.5`
- `verbosity_index = 0.5`
- `risk_tolerance = 0.5`
- `formality_score = 0.5`
- `recent_interaction_buffer = []`

当前代码中可通过后端 `resetPersonaState` 完成此操作（在全量清理逻辑中已调用）。

### 2) 不开放自定义编辑

信号规则当前内置在 `personaNode`，并且作为只读常量固化，不从外部配置文件动态加载。  
目的：防止人格核心行为被运行时随意改写，保持长期陪伴人格的一致性和稳定性。

---

## 激素（人格指标）如何被调节

每轮流程：

1. 接收用户输入  
2. 用规则识别用户信号  
3. 按命中信号对指标做增减（并 clamp 到 `[0,1]`）  
4. 生成本轮 PersonaPolicy（采样、工具、风格、记忆）  
5. 再由后续节点执行

### A. 信号到指标增量

#### verbosity（冗长度）

- 命中“详细/展开/步骤”等：`+0.08`
- 命中“简短/一句话/少说”等：`-0.08`

#### autonomy（自主性）

- 命中“你决定/直接做/不用问”等：`+0.06`
- 命中“先确认/先问我/谨慎”等：`-0.06`

#### risk（风险偏好）

- 命中“大胆/激进/尝试”等：`+0.06`
- 命中“稳妥/别冒险/安全优先”等：`-0.06`

#### formality（正式度）

- 命中“正式/专业/严谨”：`+0.06`
- 命中“随意/口语化/轻松”：`-0.06`

补充：每个类别每轮最多采纳一个信号，避免单轮输入造成过度抖动。

---

## 每个参数变化依据（从激素到策略）

## 1) 采样参数（影响 LLM 发言风格）

- `temperature = clamp(0.45 + risk*0.4 + autonomy*0.12 - formality*0.1, 0.2, 1.2)`
- `topP = clamp(0.72 + risk*0.24, 0.6, 0.98)`
- `maxTokens = clamp(520 + verbosity*980, 420, 1800)`

依据：

- 风险偏好越高，探索性越强（temperature/topP 上升）
- 正式度越高，随机性略降（temperature 下调）
- 冗长度越高，输出容量越大（maxTokens 上升）

## 2) 工具策略（影响是否执行工具）

- `confirmBeforeSensitiveTools = autonomy < 0.4 || risk < 0.4`
- `allowRiskyTools = risk >= 0.45`
- `exploratoryBias = clamp(risk*0.6 + autonomy*0.4, 0, 1)`

依据：

- 自主性或风险偏好过低 -> 敏感工具需要用户确认
- 风险偏好不足 -> 阻断高风险工具调用

## 3) 风格策略（影响语气与表述结构）

- `verbosity >= 0.65` -> `detailed`
- `verbosity <= 0.35` -> `brief`
- 其他 -> `balanced`

- `formality >= 0.65` -> `formal`
- `formality <= 0.35` -> `casual`
- 其他 -> `neutral`

再结合 autonomy 生成“主动推进 / 关键点确认”的风格指令注入上下文。

## 4) 记忆策略（影响短期与压缩行为）

- `compressThreshold = clamp(8 - verbosity*4, 4, 8)`
- `shortTermLimit = clamp(6 + verbosity*4, 6, 10)`

依据：

- 内容越详细，越容易触发压缩（避免历史膨胀）
- 同时保留更宽短期窗口（避免细节丢失）

---

## 设计解释（为什么这样做）

- `verbosity` 步长更大（±0.08），因为它对对话体感影响最直接。
- 其他维度步长设为 ±0.06，追求“可感知变化 + 不突变”之间的平衡。
- 通过 clamp 与分档阈值（0.35/0.65）保证行为连续又有阶段特征。

本质上，这是“输入信号 -> 激素调节 -> 策略输出 -> 行为表现”的闭环。
