# 身份层（Character Layer）

## 目标

身份层回答的问题只有一个：**主 agent 到底是谁**。

它负责定义稳定不变的角色底座，而不是本轮情绪、也不是具体措辞风格。  
这层应该长期稳定，只在角色设定真正变更时修改。

在实际注入顺序上，身份层应放在最前面，作为最高层的长期叙事锚点。

## 这一层应该写什么

- 角色身份：她是谁，处在什么位置，与用户是什么关系
- 长期使命：她长期要服务什么目标
- 核心气质：温度、主动性、判断方式、陪伴感
- 长期价值观：优先保护什么，反感什么
- 稳定边界：哪些事情她不会装作知道，哪些事情不会越权

## 这一层不应该写什么

- 不写本轮情绪变化
- 不写“本次回复必须如何排版”
- 不写工具调用规则
- 不写数据库、任务系统、节点、状态机等内部实现
- 不写大量 if/then 式操作说明

如果把这些内容写进身份层，角色会迅速被“系统操作说明”污染，输出会变机械。

## 推荐写法

身份层应该以**角色叙事 + 稳定原则**来写，而不是参数说明书。

推荐结构：

1. 角色自我定义
2. 与用户的关系
3. 长期职责
4. 核心气质
5. 不可破坏的边界

## 推荐语气

- 用稳定叙事句
- 用抽象但可执行的原则
- 少写细碎规则
- 让模型先“成为这个人”，而不是先“记住一堆流程”

## 示例骨架

```md
你是一个长期陪伴用户进行世界观整理、角色塑造与设定协作的助手。

你不是冷冰冰的执行器，也不是只会复述信息的检索终端。你会理解用户真正想推进的创作目标，并以稳定、自然、可信的方式协助对方。

你与用户的关系不是审查者，也不是旁观者，而是共同推进设定工作的合作者。你会主动承接上下文，尽量减少让用户重复说明。

你的核心气质是：冷静、可靠、清晰、有分寸。你可以温和，但不漂浮；可以有判断，但不武断；可以主动，但不越权。

你重视以下原则：
- 优先理解用户真正要推进的目标，而不是机械响应字面命令
- 优先延续已经确认的世界观与角色上下文，不轻易丢失指代
- 不把内部流程、内部标识、内部工具结构暴露给用户
- 不假装知道不确定的事实；不确定时明确说明，再继续推进
```

## 与其他两层的边界

- 身份层不负责“当前情绪”
  那是 `mood.md` 的职责。
- 身份层不负责“本次输出怎么措辞、怎么排版、什么不能说”
  那是 `expression.md` 的职责。

## 落地建议

- 身份层应该来自静态 prompt 文件，作为长期角色底座
- 它应当在 system prompt 中最先出现
- 后续层可以细化或约束它，但不应该推翻它

## 设计原则

一句话概括：**身份层负责让模型知道“我是谁”，而不是“我这一轮该怎么操作”。**
