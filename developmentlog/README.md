# Developmentlog 文档目录

> 状态说明（2026-04-02）
>
> 本文件用于给 AI 和开发者提供阅读指引：先读什么、每份文档解决什么问题、遇到冲突时以什么为准。

## 阅读顺序

当你第一次接手这个项目，建议按下面顺序阅读：

1. `developmentlog/AIagent-design.md`
2. `developmentlog/missiontodo.md`
3. 再进入代码阅读

原因：

- `AIagent-design.md` 说明当前已经落地并正在运行的 AI 系统架构
- `missiontodo.md` 说明当前还要继续推进的任务
- 文档看完后，再去代码中验证实现细节，效率最高

---

## 文档说明

### 1. `AIagent-design.md`

作用：

- 当前 AI 系统的架构说明书
- 描述当前已经实现的主 agent / 子 agent / 任务系统 / 消息队列 / 通信协议 / 生命周期 / 恢复链

你应该在什么情况下读它：

- 想理解当前项目怎么运行
- 想知道主 agent 和子 agent 的职责边界
- 想知道消息列表、event、turn、task、execution、notification 之间的关系
- 想知道当前代码结构应该如何解释

它回答的问题：

- 当前系统的主链是什么
- 主 agent 如何消费消息
- 子 agent 如何被创建、执行、回报
- task / execution / notification / turn 分别是什么
- 当前协议和恢复链如何工作

注意：

- 这份文档应当只描述“当前已落地事实”
- 不写未来规划、任务清单、差距分析
- 如果文档与代码冲突，应以代码中的真实运行逻辑为准，然后再修正文档

---

### 2. `missiontodo.md`

作用：

- 当前开发任务清单
- 记录下一步要推进的改造项、优化项和 UI 工作项

你应该在什么情况下读它：

- 想知道现在接下来该做什么
- 想知道当前有哪些重构方向或待办事项
- 想确认某项功能是不是“已经实现”还是“计划中”

它回答的问题：

- 当前项目下一步有哪些任务
- 哪些任务已经做过一部分
- 哪些任务还没有开始
- 后续要优先推进哪些方向

注意：

- 这份文档不是当前架构真相
- 它描述的是“后续任务”，不是“当前实现事实”
- 不要把 `missiontodo.md` 里的目标态误认为已经落地

---

## AI 阅读规则

如果你是 AI，在处理这个仓库时请遵守下面规则：

1. 先读 `AIagent-design.md`，再读 `missiontodo.md`
2. 阅读文档后，必须再去代码里核对关键链路
3. 解释“当前系统”时，以 `AIagent-design.md + 当前代码` 为准
4. 解释“接下来要做什么”时，以 `missiontodo.md` 为准
5. 如果文档和代码不一致：
   - 先以代码为运行真相
   - 再指出文档需要更新
